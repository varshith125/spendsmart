# 🚀 SpendSmart — Personal Finance Assistant

SpendSmart is a premium personal finance application designed to help you track expenses, analyze spending habits using AI, and manage your wealth effectively.

## ✨ Features
- **📊 Real-time Dashboard**: Track your current balance, budget utilization, and daily spending.
- **✨ AI Insights (Gemini)**: Personalized financial strategies generated by Google Gemini AI based on your actual spending.
- **💳 Credit Card Tracker**: Monitor multiple credit cards, track utilization, and log payments/charges.
- **🏦 Loan & Savings Tracker**: Track your loans, EMIs, and monthly auto-savings.
- **📅 Monthly Reports**: Comprehensive monthly breakdowns with PDF export functionality.
- **🔄 Recurring Transactions**: Manage subscriptions and automated bills easily.

## 🚀 Deployment Guide (Crystal Clear Steps)

Follow these steps to get SpendSmart live on the web!

### 1. Backend (on Render.com)
1.  **Link GitHub**: Create a new "Web Service" on [Render](https://render.com) and connect your repository.
2.  **Use Blueprint**: Render will automatically detect the `render.yaml` file in the root. This fixes the directory errors.
3.  **Add Environment Variables**:
    - `MONGODB_URI`: Your MongoDB Atlas connection string.
    - `JWT_SECRET`: A random long string for security.
    - `GEMINI_API_KEY`: Your key from [AI Studio](https://aistudio.google.com/).
    - `CLIENT_URL`: The URL of your Vercel frontend (once deployed).

### 2. Frontend (on Vercel.com)
1.  **Import Repo**: Create a new project on [Vercel](https://vercel.com) and import the repository.
2.  **Root Directory**: Set the "Root Directory" to `client`.
3.  **Environment Variable**:
    - Add `VITE_API_URL` and point it to your Render backend URL (e.g., `https://spendsmart-api.onrender.com`).
4.  **Deploy**: Hit the deploy button. Everything else is pre-configured in `vercel.json`.

## 🛠️ Local Development

1.  **Clone the repo**: `git clone <your-repo-url>`
2.  **Install everything**: `npm run install-all` (from the root folder)
3.  **Run Dev Mode**: `npm run dev` (starts both frontend and backend)
